## 1. 系统概述

Quant-Trade System 是一个专业级的量化交易系统，专为 A 股市场设计。系统采用前后端分离架构，集成了 **AI 智能分析**、**市场动态分析**、**策略进化引擎**、**Workspace 工作区**等创新功能，支持实时行情推送、策略回测、风险控制和实盘交易等核心功能。

### 1.1 设计目标

- **高性能**: 支持 1000+ 数据点/秒的回测速度
- **高可用**: 服务可用性 > 99.9%
- **可扩展**: 模块化设计，支持功能插件
- **安全性**: 多层安全防护，数据加密存储
- **智能化**: AI 驱动的策略生成和市场分析
- **可定制**: Workspace 工作区支持自定义面板布局

### 1.2 技术选型

| 层级 | 技术栈 | 版本要求 | 说明 |
|------|--------|----------|------|
| 前端框架 | React + TypeScript | 18+ / 5+ | 函数式组件 + Hooks |
| UI 组件库 | Ant Design | 5+ | Bloomberg 深色主题 |
| 状态管理 | Redux Toolkit | 2+ | 全局状态管理 |
| 网格布局 | react-grid-layout | 2.2.2+ | Workspace 可拖拽面板 |
| 后端框架 | FastAPI | 0.109+ | 异步 Python 框架 |
| 数据库 | PostgreSQL + TimescaleDB | 15+ | 关系型 + 时序数据 |
| 缓存 | Redis | 7+ | 数据缓存 + 会话管理 |
| 消息队列 | Celery + RabbitMQ | 5+ | 异步任务处理 |
| AI 服务 | GLM-5 / MCP 工具 | 最新版 | 智能分析引擎 |

## 5. 安全架构

### 5.1 认证流程

```
┌──────────────┐                ┌──────────────┐
│    客户端     │                │    服务端     │
└──────┬───────┘                └──────┬───────┘
       │  1. 登录请求 (用户名/密码)      │
       │ ─────────────────────────────▶│
       │                               │ 验证凭据
       │                               │ 生成 JWT
       │  2. 返回 Token               │
       │ ◀─────────────────────────────│
       │   { access_token, refresh_token }
       │                               │
       │  3. API 请求 (Bearer Token)   │
       │ ─────────────────────────────▶│
       │                               │ 验证 Token
       │                               │ 检查权限
       │  4. 返回数据                   │
       │ ◀─────────────────────────────│
       │                               │
       │  5. Token 过期时刷新           │
       │ ─────────────────────────────▶│
       │  6. 新 Token                  │
       │ ◀─────────────────────────────│
```

### 5.2 安全措施

| 层级 | 安全措施 | 说明 |
|------|----------|------|
| 传输层 | HTTPS/TLS 1.3 | 全站加密传输 |
| 认证层 | JWT + Refresh Token | 双 Token 机制，自动刷新 |
| 认证层 | 密码加密 | bcrypt 哈希存储 |
| 授权层 | RBAC 权限控制 | 角色-权限映射 |
| API 层 | 请求限流 | 100 次/分钟 (普通) / 30 次/分钟 (交易) |
| API 层 | 参数验证 | Pydantic 严格校验 |
| API 层 | CORS 配置 | 白名单域名控制 |
| 数据层 | 敏感数据加密 | AES-256 加密存储 |
| 数据库层 | 连接池 | 防止连接泄露 |
| 数据库层 | 参数化查询 | 防止 SQL 注入 |
| 日志层 | 敏感信息脱敏 | 密码、Token 不记录 |

### 5.3 交易安全

| 措施 | 描述 |
|------|------|
| 风控检查 | 每笔交易前执行风控规则 |
| 止损止盈 | 自动触发保护机制 |
| 交易确认 | 大额交易需二次确认 |
| 异常监控 | 异常交易自动告警 |
| 模拟模式 | 默认模拟交易，实盘需额外授权 |

## 11. 内置策略

系统内置 5 种量化交易策略，位于 `backend/src/strategies/builtin/` 目录：

| 策略名称 | 类名 | 文件 | 描述 |
|----------|------|------|------|
| 双均线策略 | MovingAverageStrategy | moving_average.py | 基于快慢均线交叉产生交易信号 |
| 动量策略 | MomentumStrategy | momentum.py | 基于价格动量趋势跟踪 |
| 均值回归策略 | MeanReversionStrategy | mean_reversion.py | 价格偏离均值时反向操作 |
| RSI 策略 | RSIStrategy | rsi.py | 基于相对强弱指标超买超卖 |
| 布林带策略 | BollingerStrategy | bollinger.py | 基于布林带通道突破 |

### 11.1 策略基类

所有策略继承自 `BaseStrategy`，提供统一的接口：

```python
class BaseStrategy:
    def initialize(self, context: StrategyContext) -> None
    def on_bar(self, bar: Bar) -> Optional[Signal]
    def on_tick(self, tick: Tick) -> Optional[Signal]
    def finalize(self) -> None
```

### 11.2 AI 策略包装器

`AIStrategyWrapper` 类支持将 AI 生成的策略代码动态加载执行：

- 支持文本规则解析
- 完整技术指标计算 (MA, RSI, MACD, 布林带)
- 入场/出场条件评估
- 信号置信度计算

## 13. Workspace 工作区模块 (v2.8.0 新增)

### 13.1 功能概述

Workspace 是一个可定制的工作区模块，支持用户自定义面板布局，提供类似 Bloomberg Terminal 的多面板工作体验。

### 13.2 技术实现

| 组件 | 技术 | 说明 |
|------|------|------|
| 网格布局 | react-grid-layout 2.2.2 | 拖拽、缩放面板 |
| 布局持久化 | useWorkspacePersistence Hook | localStorage 保存 |
| 实时行情 | useRealtimeQuote Hook | WebSocket 推送 |

### 13.3 可用面板类型

| 面板类型 | 组件 | 功能 |
|----------|------|------|
| chart | ChartPanel | K线图表、技术指标 |
| table | DataTablePanel | 数据表格 |
| news | NewsPanel | 新闻资讯 |
| watchlist | WatchlistPanel | 自选股列表 |
| capitalFlow | CapitalFlowPanel | 资金流向 |
| heatmap | MarketHeatmapPanel | 市场热力图 |

### 13.4 API 更新

react-grid-layout 2.2.2 版本采用了新的 API：

```typescript
// 旧版 API (已废弃)
<GridLayout
  cols={12}
  rowHeight={40}
  margin={[12, 12]}
  draggableHandle=".header"
/>

// 新版 API (v2.8.0)
<GridLayout
  gridConfig={{
    cols: 12,
    rowHeight: 40,
    margin: [12, 12],
    containerPadding: [0, 0],
    maxRows: Infinity,
  }}
  dragConfig={{
    enabled: true,
    handle: '.panel-header',
  }}
  resizeConfig={{
    enabled: true,
  }}
  compactor={verticalCompactor}
/>
```
