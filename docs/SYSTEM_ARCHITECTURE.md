## 1. 系统概述

Quant-Trade System 是一个专业级的量化交易系统，专为 A 股市场设计。系统采用前后端分离架构，支持实时行情推送、策略回测、风险控制和实盘交易等核心功能。

### 1.1 设计目标

- **高性能**: 支持 1000+ 数据点/秒的回测速度
- **高可用**: 服务可用性 > 99.9%
- **可扩展**: 模块化设计，支持功能插件
- **安全性**: 多层安全防护，数据加密存储

### 1.2 技术选型

| 层级 | 技术栈 | 版本要求 |
|------|--------|----------|
| 前端框架 | React + TypeScript | 18+ / 5+ |
| UI 组件库 | Ant Design | 5+ |
| 状态管理 | Redux Toolkit | 2+ |
| 后端框架 | FastAPI | 0.109+ |
| 数据库 | PostgreSQL + TimescaleDB | 15+ |
| 缓存 | Redis | 7+ |
| 消息队列 | Celery + RabbitMQ | 5+ |

## 4. 数据库设计

### 4.1 数据库表概览

系统共有 **63 张数据库表**，按功能模块分类：

| 模块 | 表数量 | 主要表名 |
|------|--------|----------|
| 用户管理 | 3 | users, user_preferences, user_sessions |
| 策略管理 | 3 | strategy_versions, strategy_configs, strategy_audit_log |
| 回测分析 | 3 | factor_analyses, attribution_analyses, backtest_metrics_extended |
| 交易管理 | 3 | fills, trading_calendar, daily_trade_stats |
| 投资组合 | 4 | portfolios, portfolio_positions, portfolio_risk_metrics, portfolio_optimizations |
| 行情数据 | 5 | stock_quotes, realtime_quotes, watchlists, price_alerts, market_indices |
| 风控数据 | 4 | risk_limits, risk_events, position_limits, stop_loss_orders |
| 审计日志 | 3 | audit_logs, operation_logs, system_logs |

### 4.2 核心表关系

```
users ─────────┬──▶ strategies ──▶ strategy_versions
               │         │
```

## 5. 安全架构

### 5.1 认证流程

```
┌──────────────┐                ┌──────────────┐
│    客户端     │                │    服务端     │
└──────┬───────┘                └──────┬───────┘
       │  1. 登录请求                  │
       │ ─────────────────────────────▶│
       │                               │
       │  2. 验证凭据                   │
       │                               │
       │  3. 返回 JWT Token            │
       │ ◀─────────────────────────────│
       │                               │
       │  4. API 请求 (Bearer Token)   │
       │ ─────────────────────────────▶│
       │                               │
       │  5. 验证 Token                │
       │                               │
       │  6. 返回数据                   │
       │ ◀─────────────────────────────│
```

### 5.2 安全措施

| 层级 | 安全措施 |
|------|----------|
| 传输层 | HTTPS 加密 |
| 认证层 | JWT Token + Refresh Token |
| 数据层 | 敏感数据加密存储 |
| API 层 | 请求限流、参数验证 |
| 数据库层 | 连接池、参数化查询 |

## 7. 性能指标

| 接口类型 | 目标响应时间 |
|----------|--------------|
| 健康检查 | < 50ms |
| 简单查询 | < 100ms |
| 复杂查询 | < 500ms |

## 9. 测试数据

### 9.1 测试数据脚本
