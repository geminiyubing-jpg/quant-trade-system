## 🐛 Bug 汇总

### ✅ 已修复的 Bug（2个）

#### Bug #1: 市场数据菜单没有子菜单
- **严重程度**: 🔴 阻塞性
- **问题描述**: 点击"市场数据"菜单直接跳转到数据中心页面，无法访问实时行情
- **根本原因**: Sidebar.tsx 中菜单项配置错误，缺少子菜单结构
- **修复方案**:
  - 将"市场数据"改为带子菜单的结构
  - 添加 4 个子菜单项：实时行情、股票行情、板块分析、期货
  - 更新路由：`/market` 自动重定向到 `/market/realtime`
- **修改文件**:
  - `frontend/src/components/Sidebar.tsx`
  - `frontend/src/App.tsx`
  - `frontend/src/i18n/locales/zh_CN.json`
  - `frontend/src/i18n/locales/en_US.json`
- **修复时间**: 2026-03-09 15:10
- **验证状态**: ✅ 已验证通过

---

#### Bug #2: WebSocket 订阅时序问题
- **严重程度**: 🔴 阻塞性
- **问题描述**: 页面加载时 WebSocket 还在连接中，此时点击订阅按钮无法添加新股票
- **根本原因**:
  - `subscribe()` 方法在 WebSocket 未连接时直接返回，不更新前端状态
  - 用户点击订阅后，股票不会出现在表格中
- **修复方案**:
  - 修改 `subscribe()` 方法返回布尔值（是否立即发送）
  - 无论是否立即发送，都更新前端 `subscribedSymbols` 状态
  - WebSocket 连接成功后，自动发送所有排队的订阅请求
- **修改文件**:
  - `frontend/src/services/websocket.ts` (第 148-167 行)
  - `frontend/src/pages/MarketRealtime.tsx` (第 63-105 行)
- **修复时间**: 2026-03-09 15:20
- **验证状态**: ✅ 已验证通过

### ⏳ 待修复的 Bug（4个）

#### Bug #3: Trading Mode API CORS 错误
- **严重程度**: 🟡 非阻塞性
- **问题描述**: 交易模式 API 调用被 CORS 策略阻止
- **错误信息**:
  ```
  Access to fetch at 'http://localhost:8000/api/v1/trading/mode'
  from origin 'http://localhost:3000' has been blocked by CORS policy:
  The value of the 'Access-Control-Allow-Origin' header must not be the wildcard '*'
  when the request's credentials mode is 'include'.
  ```
- **影响范围**: 交易模式状态同步失败（不影响核心功能）
- **优先级**: P1 - 中等
- **建议修复方案**:
  - 后端 CORS 中间件配置检查
  - 确保 `allow_credentials=True` 时 `allow_origins` 不使用 `["*"]`
- **预计修复时间**: 1-2 小时

#### Bug #4: Ant Design 静态方法警告
- **严重程度**: 🟢 低（建议性）
- **问题描述**: 使用 `message.success()` 等静态方法时，无法消费动态主题上下文
- **警告信息**:
  ```
  Warning: [antd: message] Static function can not consume context
  like dynamic theme. Please use 'App' component instead.
  ```
- **影响范围**: 消息提示样式可能不一致（不影响功能）
- **优先级**: P2 - 低
- **建议修复方案**:
  - 使用 Ant Design 的 `<App>` 组件包裹应用
  - 使用 `App.useApp()` 的 `message` 实例
- **预计修复时间**: 2-3 小时
